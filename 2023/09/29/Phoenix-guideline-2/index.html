主要的配置文件（config.exs）

以下是开头注释的中文翻译：

此文件负责配置你的应用程序，其依赖于 Config 模块的帮助。

此配置文件在任何依赖项之前加载，并且仅限于此项目。

也就是说「配置」是在程序运行初期，依赖项被加载前的阶段。Config 是 Elixir 内、基于键值来保存配置的模块。

「键值对」是一种存储及表示数据的方式，其中每个元素都由两部分组成：一个键（key）和一个与之相关联的值（value）。这种表示方式非常类似于字典或映射的结构，其中键用于唯一标识值，而值则用于存储相关联的数据。

这种术语的起源主要是为了描述在计算机科学和编程中常见的数据结构和数据存储方式。在这些数据结构中，键（key）通常用于查找或检索与之关联的值（value）。例如，在哈希表中，键被用作索引来查找存储在该键下的值。在编程语言中，键值对经常用于创建关联数组、字典、散列表等数据结构。

总之，「键值对」这个术语的名称是为了强调这种数据表示方式中的两个关键部分：键和与之相关联的值，它们一起构成了数据的基本单元。

网络端点的配置

网络端点为是指在一个网络服务或应用程序中，可以通过网络访问的东西，可能是某位置或 URL（统一资源定位符）。

Config.config/3（注意大小写）是 Config 模块下用于加载配置的函数。

在本文件中，其表现为 config(root_key, key, opts)，也就是配置 :demo_app 下的 DemoAppWeb.Endpoint 。

那么对于 DemoAppWeb.Endpoint，我们主要使其：

  • URL
    • 主机（host）地址为（localhost）
  • 错误信息的渲染
    • 格式
      • HTML 信息通过 DemoAppWeb.ErrorHTML 模块
      • JSON 信息通过 DemoAppWeb.ErrorJSON 模块
    • 不通过 layout
  • PubSub 服务器
  • LiveView 配置
    • 用于对需要认证的信息进行签名操作的盐

LiveView 是 Phoenix 框架的一个新功能。

开发环境

开发环境相比于其他环境，最重要的几点是：

  • 不需要将程序编译、链接、打包
  • 不需要启动复杂的鉴权
  • 最好快点看到修改代码后的成果（启动快，如果能够重载更好）
  • 能够 Debug
  • 如果遇到错误可以快速定位到（日志要有 StackTrace）
