Phoenix 框架是用于构建网络应用的框架

Phoenix is a web development framework written in Elixir which implements the server-side Model View Controller (MVC) pattern. Many of its components and concepts will seem familiar to those of us with experience in other web frameworks like Ruby on Rails or Python's Django.

按照文档的说法， Phoenix 是一个使用 Elixir 编程语言编写的网络开发框架。如果你之前有过开发经历也可以看出来，它通过服务端渲染向客户端返回数据，同时在软件设施上采用了和RoR或Django很像的 MVC（模型-窗口-控制器）架构。

原文参见 Up and Running — Phoenix v1.7.7 (hexdocs.pm) 。

其中：

  • phx.new 创建一个 Phoenix 项目
  • app 在没有其他参数的影响下：在 ./app 下创建名为 App 的目录
  • 后面跟上一堆参数【要加上 -- 标识已告诉程序这是选项】
    • --app （后面要跟上 OTP 应用的名字）
    • --module （后面要给上模块名）
    • --unbrella 是 Unbrella 项目
    • --database Ecto 的适配器（后面跟上你有的数据库）
    • --no-ecto 项目没有关于 Ecto 的内容
    • --no-assets 项目没有前端的资源文件，等同于 --no-esbuild 以及 --no-tailwind
    • --no-esbuild 如果你的项目是不需要 Javascript 的纯 API 或是你打算手动打包前端文件的话，可以选择
    • --no-tailwind 项目不会自动安装 TailwindCSS 的相关资源与依赖
    • --no-dashboard 没有控制台（模块名 Phoenix.Dashboard）
    • --no-gettext 没有多语言支持
    • --no-mailer 没法发邮件
    • --no-html 项目不包含 HTML 文件
    • --no-live 项目没有负责前端通信的 JS 部分
